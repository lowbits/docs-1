Before you can start integrating Novu into your application, you need to create a Novu account and set up a new application in the Novu Dashboard. This guide will walk you through those steps.


<Steps>
  <Step title="Sign up for Novu">
    [Create a new account](https://dashboard.novu.co/signup) on Novu Cloud or [sign in](https://dashboard.novu.co/login) if you already have an account.
  </Step>
  <Step title="Create a new Workflow">
    Create a new workflow by clicking on the **Create Workflow** button in the Workflows page, add your desired steps and save the workflow.
  </Step>
  <Step title="Add Inbox Component (Optional)">
    The most common use case is to add the Inbox component (For In-App notifications and messaging) to your application, this will allow you to display the notifications in your application.

    <Card title="Add In-App Capabilities to Your App" icon="Bell" href="/inbox/introduction">
      Quickstart for how to implement the pre-built, ready-to-use, and customizable Inbox UI component
    </Card>

  </Step>
  <Step title="Trigger a workflow">
    Each workflow is created with a unique identifier that can be used to trigger the workflow, visit the Trigger page to view the trigger instructions.

    Each trigger consistents of 3 main parts:
    - **Workflow Identifier**: The unique identifier of the workflow to trigger.
    - **Subscriber ID**: The unique identifier of the subscriber to trigger the workflow for, this can be a user id, email or any unique identifier.
    - **Payload**: The payload to pass to the workflow, this is optional and can be used to pass additional information to the workflow.

    <CodeGroup>
        ```typescript Node.js
        import { Novu } from '@novu/node';

        const novu = new Novu(process.env.NOVU_API_KEY);

        await novu.trigger('your-workflow-identifier', {
          to: {
            subscriberId: '123456',
            email: 'EMAIL_ADDRESS',
          },
          payload: {
            name: 'John Doe',
          },
        });
        ```

        ```bash Curl
        curl -X POST 'https://api.novu.co/v1/events/trigger' \
        -H 'Authorization: ApiKey NOVU_API_KEY' \
        -H 'Content-Type: application/json' \
        -d '{
          "name": "your-workflow-identifier",
          "to": {
            "subscriberId": "123456",
            "email": "EMAIL_ADDRESS"
          },
          "payload": {
            "name": "John Doe"
          }
        }'
        ```
    </CodeGroup>
  </Step>
